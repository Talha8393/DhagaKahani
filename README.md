# ShopHub — Ecommerce Store

A full-featured demo ecommerce website with a React storefront and Express mock API. Built for learning, prototyping, and as a foundation for production integration.

> **New here?** Read the [User Guide](./USER_GUIDE.md) for step-by-step instructions on shopping, checkout, account management, and the admin panel.

---

## Features

- **Storefront** — Home page, product catalog with filters/sort/pagination, product detail with variants & reviews
- **Cart & Checkout** — Persistent cart, promo codes, multi-step checkout, mock payments
- **Auth & Account** — Register, login, profile, order history, addresses, wishlist
- **Admin Panel** — Dashboard, product CRUD, order status management
- **UX** — Responsive design, loading skeletons, empty states, toast notifications, SEO meta tags, accessibility basics

---

## Tech Stack

| Layer | Technology |
|-------|------------|
| Frontend | React 18, Vite, TypeScript, Tailwind CSS |
| State | Zustand (cart + auth persist) |
| Routing | React Router 7 |
| Backend | Node.js, Express, TypeScript |
| Database | JSON files (mock — structured like a real DB) |
| Auth | JWT (bcrypt password hashing) |
| Icons | lucide-react |

## Quick Start

### Prerequisites

- Node.js 18+
- npm

### Install & run

```bash
# Clone or navigate to the project
cd D:\Practice\Ecommerce

# Install all dependencies (root + server + client)
npm run install:all

# Start both frontend and backend
npm run dev
```

| Service | URL |
|---------|-----|
| Storefront | http://localhost:5173 |
| API | http://localhost:3001 |
| API health check | http://localhost:3001/api/health |

### Individual services

```bash
npm run dev:client   # Frontend only (port 5173)
npm run dev:server   # Backend only (port 3001)
```

### Production build

```bash
npm run build
```

## API Endpoints

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| GET | `/api/products` | — | List products (filters, sort, pagination) |
| GET | `/api/products/:id` | — | Product by ID |
| GET | `/api/products/slug/:slug` | — | Product by slug |
| GET | `/api/products/categories` | — | Category list |
| POST | `/api/auth/register` | — | Create account |
| POST | `/api/auth/login` | — | Login (returns JWT) |
| POST | `/api/auth/forgot-password` | — | Mock password reset |
| GET | `/api/auth/me` | JWT | Current user |
| GET | `/api/orders` | JWT | User's orders |
| POST | `/api/orders` | JWT | Place order |
| POST | `/api/orders/validate-coupon` | — | Validate promo code |
| GET | `/api/users/addresses` | JWT | Saved addresses |
| POST | `/api/users/wishlist/toggle` | JWT | Toggle wishlist item |
| GET | `/api/admin/dashboard` | Admin | Dashboard stats |
| GET/POST/PATCH/DELETE | `/api/admin/products` | Admin | Product management |
| GET/PATCH | `/api/admin/orders` | Admin | Order management |

## Mock Data

Seed data is generated by:

```bash
cd server
npx tsx scripts/seed.ts
```

This creates:
- **35 products** across 5 categories
- **5 users** (1 admin, 4 customers)
- **4 sample orders** in various statuses
- **4 coupon codes**

Data files live in `server/src/data/`:
- `products.json`
- `users.json`
- `orders.json`
- `coupons.json`

## Extension Points (Production)

| Feature | File(s) | Notes |
|---------|---------|-------|
| Real database | `server/src/services/db.service.ts` | Replace JSON read/write with PostgreSQL, MongoDB, etc. |
| Production API URL | `client/src/services/api.ts` | Update base URL / env variable |
| Payments | `server/src/controllers/orders.controller.ts` | Integrate Stripe/PayPal at `mockPayment` |
| Email | `server/src/services/auth.service.ts` | Wire SendGrid/SES at `forgotPassword` |
| File uploads | Admin product form | Replace image URL input with S3/Cloudinary |

Search the codebase for `// EXTENSION:` comments for all marked integration points.
